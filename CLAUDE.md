# CLAUDE.md — dotnet-binding-utils

## Project Purpose
Generates .NET Android/iOS NuGet binding libraries from native Gradle/CocoaPods packages.

## Tools Overview
| ID | Tool | Description |
|----|------|-------------|
| 01 | `Dependencies.Gradle` | Uses Gradle to download Android/Java native dependencies (AARs, JARs) into the Gradle cache |
| 02 | `Binderators.Gradle` | Scans Gradle cache, builds dependency tree, generates binding `.csproj` and `.targets` via Razor templates |
| 03 | `MetadataFetcher` | Fetches and maps native libraries to their corresponding NuGet packages |

## Build Flow
```
sh bind.sh --artifact "group:artifact:version"
  └─ dotnet cake              # writes BindingHost.props with GradleImplementation
  └─ dotnet workload install  # may require sudo on system-wide .NET installs
  └─ dotnet restore BindingHost.csproj
  └─ dotnet run BindingHost.csproj
       └─ InitializeTask
       └─ BinderateTask       # ArtifactScanner → reads POM from Gradle cache → generates .csproj
       └─ NugetTask           # dotnet workload restore + dotnet pack → outputs to nugets/
```

## Key Files
| File | Purpose |
|------|---------|
| `src/Project.net8.cshtml` | Razor template for generating binding `.csproj` files (supports net9+net10) |
| `src/libs/BindingHost/BindingHost.csproj` | Entry point; imports `*.props` incl. `BindingHost.props` |
| `src/libs/Binderator.Gradle/ArtifactScanner.cs` | Reads `.pom`/`.module` from Gradle cache to build dependency graph |
| `src/libs/Binderator.Gradle/Engine.cs` | Generates .csproj via Razor templates |
| `src/android/<group>/<artifact>/nuget.json` | NuGet package ID, name, dependencyOnly flag |
| `src/android/<group>/<artifact>/<version>.json` | Per-version config (revision, runtimeDependenciesAsCompile) |
| `src/android/<group>/<artifact>/<version>.fixed.json` | Override AndroidX/Kotlin NuGet dependency versions |
| `nugets/` | Output folder for built local `.nupkg` files |

## binding/ Directory Structure
Each artifact has a `binding/` folder whose contents are partially gitignored:
```
src/android/<group>/<artifact>/
  .gitignore         # controls which binding/ files are committed
  nuget.json
  <version>.json
  binding/
    <PackageId>.csproj    # GENERATED by Binderator — regenerated on every bind.sh run
    <PackageId>.targets   # GENERATED by Binderator — regenerated on every bind.sh run
    Additions/
      Additions.cs        # COMMITTED — hand-written C# partial classes compiled with the binding
    Transforms/
      *.xml               # COMMITTED — metadata/transforms to fix binding errors
    Extra.props           # COMMITTED (if needed) — extra MSBuild imports that survive regeneration
```

**Important**: `binding/*.csproj` and `binding/*.targets` are regenerated on every `bind.sh` run. Never make permanent edits to those files — put custom logic in `Additions.cs`, `Transforms/`, or `Extra.props` instead.

## Extra.props Pattern
When `Additions.cs` needs compile-time references that Binderator does not auto-include in the generated `.csproj`, create `binding/Extra.props`:

```xml
<Project>
  <ItemGroup>
    <PackageReference Include="SomePackage" Version="x.y.z" PrivateAssets="none" />
  </ItemGroup>
</Project>
```

The Razor template (`src/Project.net8.cshtml`) already includes:
```xml
<Import Condition=" EXISTS('Extra.props') " Project="Extra.props" />
```

Also add to the package `.gitignore`:
```
!binding/Extra.props
```

## Known Issues / Gotchas
- **`dotnet workload restore` requires sudo** on system-wide .NET (`/usr/local/share/dotnet/`). Workaround: run `dotnet pack` directly after BinderateTask generates the `.csproj`.
- **ArtifactScanner requires `.pom` or `.module` in Gradle cache** (`~/.gradle/caches/modules-2/files-2.1/`). If only AAR is cached (no POM), dependencies won't be discovered → generated `.csproj` missing `PackageReference` items.
- **Gradle metadata** (`descriptor.bin` in `modules-2/metadata-2.106/`) contains dependency info but ArtifactScanner doesn't read it.
- **Mapbox packages must be built bottom-up**: each package needs its NuGet dependencies available before it can be built.
- **Root-owned `obj/` and `bin/`** after a `sudo`-run build block subsequent non-sudo builds with `GDL004: Access denied`. Fix: `sudo chown -R $(whoami) src/libs/BindingHost/obj src/libs/BindingHost/bin`
- **Gradle `.gradle` folder locked**: Kill Java process first (`taskkill -F -im java.exe` on Windows).
- **CS0738 IParcelableCreator (.NET 9/10)**: Generator emits `Creator` classes with non-nullable `CreateFromParcel`/`NewArray` returns, but the interface now requires `Object?`/`Object[]?`. Fix: add explicit interface impl in Additions.cs partial — `Java.Lang.Object? IParcelableCreator.CreateFromParcel(Parcel? s) => CreateFromParcel(s);`
- **Generator type-param `classBound` resolution (silent class rejection)**: The generator independently validates each type parameter's `classBound` against `--ref` DLLs — separate from class-parse. If the bound type is absent (e.g. `Layer` dropped by maps-style-ndk27's class-parse), the generator silently drops the class AND all `<add-node>` subclasses. Symptom: classes in `api.xml.fixed` but absent from `generated/src/`. Fix: change `classBound` to `java.lang.Object` and remove `genericConstraints` — see Metadata Snippets below.
- **`<add-node>` + abstract C# base**: Classes restored via `<add-node>` that extend a generated `abstract` C# class must implement every abstract member in Additions.cs using `_members.InstanceMethods.InvokeVirtualObjectMethod`/`InvokeVirtualVoidMethod`. Symptom: CS0534 "does not implement inherited abstract member".
- **NU1504 duplicate in Extra.props**: Binderator auto-adds POM transitive dependencies to the generated `.csproj`. Adding the same package in `Extra.props` causes NU1504. Verify the generated `.csproj` before adding references to `Extra.props`.
- **Full rebuild with `bind-mapbox-ndk27.sh`**: BinderateTask runs (regenerates all .csprojs) but NugetTask always fails due to workload permissions. Follow with `dotnet pack <binding/*.csproj> -c Release -o nugets/` for each package in build order. After deleting nugets, also re-run `sh bind.sh --artifact "com.mapbox.common:common-ndk27:24.15.3"` first — its .csproj may have drifted to the previous version.
- **Root-owned binding `obj/` directories** block builds with XARDF7019 (UnauthorizedAccessException on `generated/type-mapping.txt`). Move them entirely OUT of the source tree — do NOT rename to `obj.bak/` in the same folder; javac scans sibling dirs and produces duplicate-class JAVAC0000 errors. Safe fix: `find src/android -maxdepth 5 -name "obj" -type d | while read d; do mv "$d" /tmp/; done`
- **`dotnet pack` output truncation**: The Bash tool caps output at ~364 KB. If the pack result is cut off, check whether the DLL compiled by looking in `binding/bin/Release/net10.0-android/`. If the DLL is there, run `dotnet pack --no-build -c Release -o nugets/` to skip the build step and just produce the `.nupkg`.
- **Manual `.csproj` version bump** (when `bind.sh` is unavailable due to root-owned BindingHost/obj/): find the Gradle cache hash for the new version with `ls ~/.gradle/caches/modules-2/files-2.1/<group>/<artifact>/<version>/`, then edit the `.csproj` using `replace_all` for old-hash → new-hash, old `<version>/` path segments → new, and all version string fields (`PackageVersion`, `FileVersion`, `AssemblyVersion`, `InformationalVersion`, `Summary`, `Description`).
- **New `<version>.fixed.json` needed for each artifact version**: Binderator does not inherit `fixed.json` across versions — copy from the previous version's file when bumping to a new release.
- **`.gitignore` audit for `Extra.props`**: Every package with a committed `binding/Extra.props` must have `!binding/Extra.props` in its `.gitignore`. Audit with: `find src/android -name "Extra.props" | while read f; do pkg=$(dirname $(dirname $f)); grep -q "Extra.props" $pkg/.gitignore || echo "MISSING: $pkg"; done`

## BG8700/BG8800 — Transitive Type Resolution
The binding generator (`generator` tool) validates types **transitively** from `--ref` DLLs. When `base-ndk27.dll` contains `IMapCameraManagerDelegate.SetCamera(CameraOptions)`, the generator looks up `CameraOptions` in all `--ref` DLLs. If `android-core-ndk27.dll` is not in the ref list, BG8700/BG8800 fires even though the binding's own `api.xml` doesn't mention `CameraOptions` at all.

**Standard Extra.props recipe for any ndk27 plugin that references `base-ndk27`:**
```xml
<!-- Delegate interfaces (IMapCameraManagerDelegate etc.) are in base-ndk27 -->
<PackageReference Include="Com.Mapbox.Maps.BaseNdk27" Version="11.15.3" PrivateAssets="none" />
<!-- Camera types (CameraOptions, ScreenCoordinate, Image, etc.) are in android-core-ndk27 -->
<PackageReference Include="Com.Mapbox.Maps.AndroidCoreNdk27" Version="11.15.3" PrivateAssets="none" />
<!-- Cancelable, LocationError, IGeofencingUtils are in common-ndk27 -->
<PackageReference Include="Com.Mapbox.Common.CommonNdk27" Version="24.15.3" PrivateAssets="none" />
<!-- Point, Feature, IGeometry are in mapbox-sdk-geojson -->
<PackageReference Include="Com.Mapbox.Mapboxsdk.MapboxSdkGeojson" Version="7.5.0" PrivateAssets="none" />
```

**Debugging BG errors — key artifacts to inspect:**
- `binding/obj/Release/net10.0-android/generated/generator.rsp` — lists all `--ref` DLLs the generator sees; missing refs here = BG errors
- `binding/obj/Release/net10.0-android/generated/type-mapping.txt` — maps Java types → C# types for a given package; use to find which package owns a type
- `binding/obj/Release/net10.0-android/java-resolution-report.log` — shows types removed from api.xml during class-parse (class-parse has no reference JARs, only the InputJar)

## Metadata Snippets for Common Binding Errors
Use browser DevTools console to generate metadata XML from error output:

### Duplicate field/method — rename managed member
```js
var x = `PASTE_ERRORS_HERE`;
console.log(x.split('\n')
  .map(item => /'([^']+)'/.exec(item)?.[1]).filter(Boolean)
  .map(item => /.(\w+)$/.exec(item)[1])
  .map(item => item[0].toLowerCase() + item.substring(1))
  .map(item => `<attr path="//field[@name='${item}']" name="managedName">${item}</attr>`)
  .join('\n'))
```

### Cannot change access modifiers when overriding — set visibility to protected
```js
var x = `PASTE_ERRORS_HERE`;
console.log(x.split('\n')
  .map(item => /'([^']+)'/.exec(item)?.[1]).filter(Boolean)
  .map(item => /(\w+).(\w+)\((.+)?\)/.exec(item)).filter(Boolean)
  .map(item => `<attr path="//class[@name='${item[1]}']/method[@name='${item[2][0].toLowerCase()+item[2].substring(1)}' and count(parameter)=${item[3]?item[3].split(',').length:0}]" name="visibility">protected</attr>`)
  .join('\n'))
```

### Return type must be 'Object' — set managedReturn
```js
var x = `PASTE_ERRORS_HERE`;
console.log(x.split('\n')
  .map(item => /'([^']+)'/.exec(item)?.[1]).filter(Boolean)
  .map(item => /(\w+).(\w+)\((.+)?\)/.exec(item)).filter(Boolean)
  .map(item => `<attr path="//class[@name='${item[1]}']/method[@name='${item[2][0].toLowerCase()+item[2].substring(1)}' and count(parameter)=${item[3]?item[3].split(',').length:0}]" name="managedReturn">Java.Lang.Object</attr>`)
  .join('\n'))
```

### Companion field conflict
```xml
<attr path="//field[@name='Companion']" name="managedName">CompanionField</attr>
```

### Type parameter `classBound` unresolvable — relax to Object
When a class (or its `<add-node>` subclasses) is silently absent from `generated/src/` despite appearing in `api.xml.fixed`, check if its type parameter `classBound` references a type dropped by class-parse:
```xml
<attr path="//class[@name='Foo']/typeParameters/typeParameter[@name='T']"
      name="classBound">java.lang.Object</attr>
<attr path="//class[@name='Foo']/typeParameters/typeParameter[@name='T']"
      name="jni-classBound">Ljava/lang/Object;</attr>
<remove-node path="//class[@name='Foo']/typeParameters/typeParameter[@name='T']/genericConstraints" />
```
